## A Process That Resonates

<p><a href="http://processing.org/">Processing</a> is a free open-source programming language for people who want to program images, animation, and interactions. It was created with artists, designers and hobbyists in mind.</p>

<p>After seven years of alpha and beta testing <a href="http://reas.com/blog/archives/57">version 1.0 of Processing was launched yesterday</a>.</p>

<p>I first discovered Processing in late 2003 and started coding sketches in <a href="http://stungeye.com/archives/2004/02/ordered_chaos_r.php">February of 2004</a>. Since then I&#8217;ve logged countless hours geeking out with this wonderful tool. I published <a href="http://stungeye.com/programs/">my early sketches</a> online, but I think the last sketch I shared on StungEye was a <a href="http://stungeye.com/processing/1007/">crystal growth simulator</a> back in October of 2007.</p>

<p>Lately when I want to do some experimental coding I pull out my <a href="http://shoooes.net/">Ruby Shoes</a>, but I <em>have</em> started teaching some of my students how to write programs for their cellphones using the <a href="http://mobile.processing.org/">mobile version of Processing</a>.</p>

<p>I also own an <a href="http://www.arduino.cc/">Arduino</a>, the electronics-hacking version of Processing. Next week Andrew and I are going to use an Arduino to build a <a href="http://en.wikipedia.org/wiki/Random_number_generator">random number generator</a>. Why? Well, for use as a <a href="http://video.google.com/videoplay?docid=4148462560022725286&amp;hl=en">consciousness field resonator</a> to test the existents of the <a href="http://en.wikipedia.org/wiki/Noosphere">Noosphere</a>, of course. :)</p>

<p>BTW, if you want to see what&#8217;s possible with Processing check out the work of <a href="http://www.flight404.com/blog/">Robert Hodgin</a> or <a href="http://vimeo.com/1747316">Glenn Marshall</a>.</p>

<p>If you want to learn Processing you can <a href="http://processing.org/learning/gettingstarted/index.html">get started</a> by following <a href="http://processing.org/learning/tutorials/">the tutorials</a> and <a href="http://processing.org/reference/index.html">the API reference</a>. You can learn from and be inspired by <a href="http://openprocessing.org/">other</a> peoples <a href="http://processing.org/exhibition/index.html">code</a>. I also recommend the book <a href="http://www.learningprocessing.com/">Learning Processing</a> by <a href="http://www.shiffman.net/">Daniel Shiffman</a>.</p>
