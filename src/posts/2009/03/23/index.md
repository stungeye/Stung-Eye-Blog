<h3>Sunday - Toolkit Overviews and Dorkbot @ Brillobox</h3>

<p>Sunday was a day of presentations. From 8:45 to 17:45 we were treated to 45 minute demos of various arty programming toolkits. In the evening there was a <a href="http://dorkbot.org">Dorkbot</a> <a href="http://www.pecha-kucha.org/">Pecha Kucha</a>-ish gathering at a pub called the <a href="http://www.flickr.com/photos/juli/64299189/">Brillobox</a>.</p>

<p>Some notes I jotted down during the presentations:</p>

<h4>Alice - Presented by CMU’s <a href="http://www.cs.cmu.edu/~djslater/">Don Slater</a>.</h4>

<p><a href="http://www.alice.org/">Alice</a> is a 3D programming environment for creating animations, games and videos. 3d models of objects and characters are programmed visually by dragging/dropping/connecting various Instruction-Tiles together.</p>

<p><em>Jotted Down:</em></p>

<p>Five Fundamentals of Programming</p>

<ul>
<li>Sequence</li>
<li>Decision</li>
<li>Repetition</li>
<li>Data Persistence (Variables)</li>
<li>Methods (Modular Abstractions)</li>
</ul>
<h4>Pd - Presented by <a href="http://at.or.at/hans/blog/">Hans-Christoph Steiner</a>
</h4>

<p><a href="http://puredata.info/">Pd</a> or Pure Data is a graphical programming language for audio and video programming. Data (most often audio signals) flows through various specialized transformation or generation nodes/objects which are visually connected together by the programmer using “wires”.</p>

<p>1000 words == <a href="http://footils.org/tut/pddrums/drumcenter-01.png">Picture</a></p>

<p><em>Jotted Down:</em></p>

<p>Closed tools split creators from consumers.</p>

<p>We need to think about programming as something every does / will do.</p>

<h4>Scratch - Presented by <a href="http://web.media.mit.edu/~jmaloney/">John Maloney</a> and <a href="http://evelyn.aquapixel.net/">Evelyn Eastmond</a>
</h4>

<p><a href="http://scratch.mit.edu/">Scratch</a> is a graphical programming environment for children ages 8 and up. Like Alice, the programs are created by snapping graphical blocks together.</p>

<p><em>Jotted Down:</em></p>

<p>Remixing of Scratch programs is encourage via their website.</p>

<p>There appears to be a nice learning path from Scratch through Processing to Java. Perhaps we could use Scratch as an introductory programming language at the college. <a href="http://www.cs.harvard.edu/~malan">David Malan</a> has been using it for a <a href="http://www.cs50.net/">intro comp sci course at Harvard</a>.</p>

<p>Scratch is implemented in <a href="http://www.squeak.org/">Squeak Smalltalk</a>.</p>

<p>May 16th is <a href="http://day.scratch.mit.edu/">Scratch Day</a>.</p>

<p><a href="http://scratch.mit.edu/projects/MahoAshley/101648">Scratch anime</a>!</p>

<h4>Hackety Hack - Presented by <a href="http://whytheluckystiff.net/">_why</a>
</h4>

<p><a href="http://github.com/why/hacketyhack/tree/master">Hackety Hack</a> is a Ruby-based environment designed to make programming accessible for kids, (especially teenagers). Its motivation: <a href="http://whytheluckystiff.net/articles/theLittleCodersPredicament.html">The Little Coder’s Predicament</a></p>

<p><em>Jotted Down:</em></p>

<p>Why claims to be a freelance professor, in other words, half-way between a scarecrow and a seeing eye dog.</p>

<p>A logic/programming card game called <a href="http://www.flickr.com/photos/schmarty/3344882147/">kaxXxt</a> is also in the works.</p>

<p>The ASCII sword-fighting program stirs-up long dormant memories of <a href="http://en.wikipedia.org/wiki/Monkey_Island_%28series%29">Monkey Island</a>.</p>
